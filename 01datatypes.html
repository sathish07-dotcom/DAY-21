datatypes - in- javascript

In JavaScript, there are several data types that are used to store different kinds of data.
These can be divided into **primitive types** and **non-primitive (or reference) types**.

Primitive Data Types:

  • **Number**: Represents both integer and floating-point numbers.

    Example: `let age = 25;` or `let price = 19.99;`

  • **String**: Represents a sequence of characters enclosed in single, double, or backticks (for template literals).

    Example: `let name = "John";` or `let greeting = `Hello, ${name}`;`

  • **Boolean**: Represents a value that is either `true` or `false`.

    Example: `let isActive = true;`

  • **Undefined**: A variable that has been declared but not yet assigned a value.

    Example: `let x; // x is undefined`

  • **Null**: Represents the intentional absence of any object value.

    Example: `let person = null;`

  • **Symbol** (ES6): Represents a unique and immutable value, often used as object keys to avoid name clashes.

    Example: `let uniqueSymbol = Symbol("id");`

  • **BigInt** (ES11): Represents integers with arbitrary precision.

    Example: `let bigNumber = 1234567890123456789012345678901234567890n;`

    • Non-Primitive (Reference) Data Types:

  • **Object**: A collection of key-value pairs, where the keys are strings and the values can be any data type.

    Example: `let person = { name: "John", age: 25 };`

  • **Array**: A special type of object used for storing ordered collections of values.

    Example: `let numbers = [1, 2, 3, 4];`

  • **Function**: A block of code that can be called and executed. In JavaScript, functions are also objects.

    Example: function greet(name) { return `Hello, ${name}!`; }
